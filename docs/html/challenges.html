Challenges¶

Non-EVM Chains¶

A fork that is deployed on a non-EVM chain such as Invictus DAO are a challenge as the same code cannot be used to collect metrics. Special pipeline steps will be required to deal with non-EVM forks. As long as the basic metrics can be pulled then the extended metrics should be able to be calculated just as with EVM forks.

Non-forks¶

Non-forks are on EVM but are not really forks. And example of this would be Temple. Again, this will likely require special pipeline steps to grab the basic metrics. As with non-EVM chains, as long as the basic metrics can be pulled then the extended metrics should be able to be calculated.

Close Source¶

Closed source or just not publishing the code base on Github or another public source code repository leads to issues trying to track each of the contracts down. As an example, most of the OHM forks if they replicate the OHM docs then obtaining the contracts is pretty simple. But there are a few contracts that are not included in the docs and those really require access to github repos to find. These forks will either need to have missing metrics or if the devs are helpful they might be able to provide.
